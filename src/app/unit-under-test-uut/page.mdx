## Overview

Tracking hardware tests by serial number is key for traceability and performance monitoring. With TofuPilot, you can specify serial numbers, part numbers, revisions, and batch numbers for each unit. This makes it easy to trace test results, spot quality issues, and analyze performance across batches or revisions.

## Integration

With frameworks like OpenHTF, the serial number and additional parameters can be added as metadata when `Test` is specified. For custom test frameworks using the Python client, specify parameters in `unit_under_test` when creating the run.

    <Property name="unit_under_test" type="array">
        Specify information of the unit being tested.

    <Property name="serial_number" type="str"  indent={1}>

    <Property name="part_number" type="str"  indent={1}>

<Note>

## In-app view

You can find infromation about your Unit Under Test in the **Unit** section of the **Run** page.

<Image src="/uut-page-run.png" alt="Unit Under Test (UUT) on a Run page." />

Clicking on the unit will take you to the dedicated **Unit** page, where you can view details about the component, including the part number, name, and revision.

<Image src="/uut-page-unit.png" alt="Unit Under Test (UUT) on a Unit page." />

These features will be explained in more detail in the [Traceability](/traceability) section.

### Rename a component

You can rename a Component's name, a Component number (`part_number`), or `revision` directly in the application. Go to the Component page, then select **Edit name**, **Edit part number**, or **Edit revision** from the menu.
